# 项目级 AGENTS 规则

本文件用于约束代理在当前仓库中的改动方式。目录级或更下层的 `AGENTS.md` / `AGENTS.override.md` 如存在，优先级高于本文件。

## 项目边界

- 当前仓库只维护通达信 TQ Python 的非交易基础接口脚本、路径诊断与文档记录。
- 不推进交易相关接口开发，除非用户明确改变项目边界。
- 不修改 `.agents/skills/tdx-quant/SKILL.md`。
- 不把 `tq.price_df` 这类进程内 helper 强行包装成独立 CLI。
- 不为“以后可能会用”提前引入通用框架、插件系统或多层抽象。

## 文件落点

- `scripts/` 只放对外命令入口。
- 原子接口命令继续平铺在 `scripts/` 根目录。
- 共享逻辑一律优先下沉到 `lib/`。
- `lib/config.py` 只保留兼容出口，不再新增实际逻辑。
- 路径发现、配置存取、路径诊断相关逻辑优先放 `lib/pathing/`。
- 输出、运行时初始化、脚本公共适配逻辑优先放 `lib/cli/`。

## 文档分工

- 命令入口和常用用法写到 `scripts/README.md`。
- 接口测试结论、兼容性与实测行为写到 `docs/interfaces/TEST_STATUS.md`。
- 边界、目录结构、模块开发约定写到 `ARCHITECTURE.md`。

## 本地配置

- `lib/tdx_config.json` 是机器私有配置文件。
- 该文件保留在本机，不提交到仓库。

## 改动与验证

- 改 CLI 时，优先跑最小范围的 CLI 测试。
- 当前建议验证入口：
  - 改统一入口时，优先跑 `tests.test_unified_cli`。
  - 改 CLI 参数与帮助信息时，优先跑 `tests.test_cli_help`。
  - 改路径发现或配置时，优先跑 `tests.test_pathing_discovery`、`tests.test_config_store`。
- 如果测试目录或命名调整，改动时同步更新本文件。
- 优先最小必要验证，不默认跑高成本全量命令。

## 工作方式

- 先复用现有模式，再考虑新增结构。
- 一次改动只解决当前问题，不顺手做无关重构。
- 真实行为与版本差异先写进文档，再决定是否继续抽象。
- 遇到正式版与 mock 版差异时，优先做兼容判断，不先改公共命令名。

## 结构化与拆分阈值

- `scripts/*.py` 默认保持薄入口，只做参数解析、调用内部能力和输出。
- 如果脚本入口开始承载大段业务逻辑、路径处理、序列化或状态编排，应优先下沉到 `lib/`。
- 单文件超过 `300` 行且同时混合多类职责时，优先考虑拆分。
- 新增共享逻辑只有在至少两个明确调用点出现时，才考虑抽成共享模块。
